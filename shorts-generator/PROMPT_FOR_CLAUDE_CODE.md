CLAUDE.md를 먼저 읽고 시작해줘. 아래 작업을 순서대로 진행하되, 각 단계 전에 변경 계획을 짧게 요약해서 보여주고, 확신 없는 API 세부사항은 절대 지어내지 말고 "미검증"이라고 표시하거나 공식 문서를 검색해서 확인한 뒤 구현해줘.

## 배경
"만약에" 시리즈 유튜브 쇼츠 자동 제작 웹앱에 아래 기능들을 추가하려고 해. 현재 파이프라인은 주제입력 → 대본생성 → 이미지생성 → 나레이션 → 합성 순서로 동작하고 있어.

## 작업 1: 대본 생성에 오프닝 훅 규칙 + 아웃트로(구독유도) 씬 추가

`backend/src/services/claude.js`의 `generateScript` 함수 수정:

1. **오프닝 씬 규칙 강화**: 첫 번째 씬의 imagePrompt는 반드시 "시대착오적 오브젝트(예: 조선시대 인물 + 스마트폰)가 프레임 중심에 오고, 인물의 표정(놀람/신남/당황)과 동작(뭔가 하려는 찰나)이 명확히 드러나도록" 지시하는 규칙을 system 프롬프트에 추가해줘. 정지된 구도가 아니라 "사건이 벌어지는 중"처럼 보이게.

2. **아웃트로 씬 자동 추가**: 마지막 씬 다음에 별도의 아웃트로 씬을 하나 더 생성해줘.
   - narration: 자연스러운 구독 유도 문구 + 다음 편 예고 (예: "이런 반전 더 보고 싶다면 구독 꾹! 다음엔 [다른 시대/소재] 편으로 찾아올게요" — 실제 문구는 Claude가 매번 다르게 생성하도록 프롬프트에 "다음 편 소재는 아직 안 정해졌으니 궁금증을 유발하는 톤으로만 예고" 지시)
   - imagePrompt: 본편과 스타일은 통일하되, 구독 버튼이나 화살표 등 시각적 유도 요소를 암시하는 구도
   - JSON 응답 스키마에 `sceneType: "content" | "outro"` 필드 추가

3. **DB 스키마 반영**: `backend/src/db/init.js`의 `scenes` 테이블에 `scene_type TEXT DEFAULT 'content'` 컬럼 추가. 기존 DB 파일이 있으면 마이그레이션 안내(ALTER TABLE 스크립트나 재생성 안내)도 같이 작성해줘.

4. **라우트 반영**: `backend/src/routes/pipeline.js`의 프로젝트 생성 로직에서 scene_type도 함께 저장하도록 수정.

## 작업 2: 프론트엔드 — 아웃트로 씬 시각적 구분

`frontend/src/pages/ImagePage.jsx`와 `NarrationPage.jsx`에서 아웃트로 씬은 카드에 "아웃트로" 뱃지를 표시해서 본편과 구분되게 해줘. 기존 CSS 클래스 네이밍 컨벤션(`index.css`)을 따라줘.

## 작업 3: 큐레이션 체크리스트 (정책 준수용 최소 개입 장치)

`RenderPage.jsx`에 렌더링 버튼을 누르기 전 체크박스 2개를 추가해줘:
- "이미지 재생성 여부를 확인했습니다"
- "자막/나레이션 내용을 확인했습니다"

둘 다 체크해야 렌더링 버튼이 활성화되도록. 이건 완전 자동화가 아니라는 걸 기록으로 남기기 위한 최소 장치야 (완벽한 정책 준수를 보장하진 않지만 최소한의 사람 개입 흔적을 남기는 목적).

## 작업 4: Ken Burns 효과 개선

`backend/src/services/ffmpeg.js` 수정:

1. **연속 중복 방지**: 현재 `randomDirection()`이 완전 랜덤이라 연속된 씬이 같은 방향(예: 줌인→줌인)으로 나올 수 있음. 이전 씬의 방향을 기억해서, 직전과 다른 방향이 나오도록 로직 수정. `imageToClip`을 호출하는 쪽(`backend/src/routes/pipeline.js`의 `/render` 라우트)에서 이전 방향을 추적하며 순회하도록 변경.

2. **오프닝 씬 줌인 고정**: `scene_order === 0`인 오프닝 씬은 랜덤 방향 대신 항상 `"zoom-in"`으로 고정. 시선을 중심 오브젝트로 끌어당기는 효과가 훅으로서 더 강하기 때문. `/render` 라우트에서 씬 순서를 확인해 오프닝만 예외 처리.
